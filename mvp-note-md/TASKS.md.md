# Task 1: 修复 iPhone Safari 权限流程

## 目标
让页面在 iPhone Safari 上更稳地请求并处理以下权限：

- camera
- motion / orientation（如果当前交互依赖）

## 需要完成
- 检查当前权限请求触发时机
- 确保权限请求由用户操作触发
- 优化权限失败时的提示文案
- 明确区分：
  - 相机权限拒绝
  - 动作权限拒绝
  - 浏览器不支持
- 保证权限失败后页面不会卡死

## 验收标准
- 用户点击开始按钮后，能正确触发权限流程
- 相机权限失败时能看到明确提示
- motion 权限失败时能看到明确提示
- 页面状态不会混乱
- README 中补充权限行为说明

## 约束
- 不重写整个交互逻辑
- 不改变当前核心捕获循环
- 不引入框架

# Task 2: 加位置权限和 geolocation 调试显示

## 目标
让页面具备最小位置能力，便于后续 geofence 测试。

## 需要完成
- 请求位置权限
- 获取当前位置
- 在页面中显示：
  - 权限状态
  - 纬度
  - 经度
  - 精度（如果可得）
- 处理以下情况：
  - 用户拒绝
  - 浏览器不支持
  - 获取超时
  - 获取失败

## 验收标准
- 页面可以请求位置权限
- 成功时能显示当前位置信息
- 失败时有清晰提示
- 不影响现有相机与捕获逻辑

## 约束
- 保持 UI 简洁
- 调试信息可以先以开发面板形式出现
- 不做地图页
- 不做后台记录

---

# Task 3: 加 geofence 测试模式

## 目标
增加一个最简单的地点触发测试机制。

## 需要完成
- 在代码中配置一个测试坐标点
- 配置一个测试半径（如 50m / 100m）
- 判断用户当前位置是否进入 geofence
- 在页面上显示 geofence 状态：
  - inside
  - outside
  - unknown
- 将 geofence 状态接入 demo 逻辑，例如：
  - 只有进入范围后蝴蝶才出现
  - 或只有进入范围后才允许捕获

## 验收标准
- 可以通过配置坐标和半径测试 geofence
- 页面能明确显示当前位置是否进入范围
- geofence 逻辑能影响体验流程
- 不引入复杂地图或后台服务

## 约束
- 先做单点 geofence
- 不做多点管理
- 不做复杂规则系统
- 不做商家/活动后台

# Task 4: 美化结果页

## 目标
提升捕获成功后的获得感和展示感。

## 需要完成
- 优化结果页视觉层次
- 更明显地区分：
  - success
  - fail
  - rare（如已有）
- 增加更好的文案和视觉反馈
- 让用户更明确感知“抓到了什么”

## 验收标准
- 成功和失败状态一眼可区分
- 结果页更有完成感
- 不影响当前主流程稳定性

## 约束
- 不引入复杂美术依赖
- 不做大型动画系统
- 保持轻量
