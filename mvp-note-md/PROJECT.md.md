

## 项目名称
ifafa-demo

## 项目目标
做一个可在 **iPhone Safari** 上直接运行的 **iButterfly-like 最小 Demo**，用于快速验证“现实背景中的发现—捕获—揭晓”这条最小体验闭环。

这个项目当前的重点不是重 AR，不是完整产品，而是验证以下问题：

1. 用户在手机相机背景下，是否会感到“真的在抓一个东西”
2. 滑动捕获和甩动捕获，哪种更顺手
3. 捕获成功后的即时反馈，是否足够形成获得感
4. 未来加入位置能力后，是否能自然过渡到地点触发玩法

---

## 当前范围

### 当前已实现功能
- 相机预览
- 漂浮蝴蝶视觉对象
- 滑动捕获
- 甩动捕获
- 成功 / 失败反馈
- 简单结果展示

### 当前技术形态
- 纯前端静态 Demo
- 运行方式：浏览器打开
- 目标环境：iPhone Safari
- 文件结构：
  - `index.html`
  - `styles.css`
  - `app.js`
  - `README.md`

---

## 下一阶段目标
下一阶段只做两件事：

1. **加 geolocation**
   - 请求位置权限
   - 获取当前位置
   - 页面上显示位置状态与经纬度
   - 让位置能力进入最小可测状态

2. **加简单 geofence**
   - 设定一个测试坐标点和半径
   - 判断用户是否进入范围
   - 进入范围后再允许蝴蝶出现或捕获
   - 用于验证“地点触发”是否成立

## 产品判断原则
这个项目的核心不是技术炫技，而是体验闭环验证。

优先级顺序如下：

1. 先保证能打开
2. 先保证权限流程可走通
3. 先保证捕获交互顺畅
4. 先保证反馈足够明确
5. 再加位置
6. 最后才考虑更复杂系统

---

## 设计原则

### 1. 保持轻量
只允许做最必要的功能，不引入不必要的依赖和复杂工程结构。

### 2. 保持纯前端
当前阶段尽量维持纯 HTML / CSS / JS，不引入打包器、框架或大型依赖，除非明确需要。

### 3. 保持可直接部署
项目应保持为静态文件可部署形式，适合 GitHub Pages、Netlify、Vercel 等静态托管。

### 4. 保持移动端优先
一切交互优先考虑 iPhone Safari，而不是桌面浏览器体验。

### 5. 保持可读性
代码应比“炫”更重要。后续要方便 Codex 和人类继续接手。

## 关键技术约束
- `getUserMedia` 需要安全上下文，部署时应使用 HTTPS
- iPhone Safari 上 motion / orientation 权限通常需要用户手势触发后请求
- 项目必须兼容 GitHub Pages 的路径结构
- 静态资源引用尽量使用相对路径，不要使用根路径绝对引用
