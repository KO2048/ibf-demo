# ifafa Demo

这是一个面向 **iPhone Safari** 的轻量静态前端 Demo，用来验证单机版“发现 - 捕捉 - 揭晓”体验闭环。

当前版本只验证本地可运行的核心玩法，不涉及地理位置或 geofence：

- 相机背景
- 3 种真实蝴蝶素材变体
- 2.5D rig 扇翅与拟态飞行
- 雷达搜索 + 锁定 + 压圈 + 时机撒网主流程
- 甩动捕捉实验模式
- 固定测试网、警觉值与结果反馈

## 文件

- `index.html`
- `styles.css`
- `app.js`

## 使用方式

1. 把整个目录部署到 **HTTPS** 静态托管
   - GitHub Pages
   - Netlify
   - Vercel
   - Cloudflare Pages
2. 用 **iPhone Safari** 打开页面
3. 点击“开始体验”
4. 允许相机权限
5. 按住屏幕拖动准星，寻找强信号并积满锁定进度
6. 蝴蝶显形后继续把它稳定留在准星内，压满追踪进度
7. 进入撒网窗口后，看准 timing 再点击“撒网”
8. 如需测试实验模式，再点击“实验模式：甩动”并允许动作权限

## 当前体验重点

- 默认演示路径是 **雷达捕捉**
- 甩动模式是次级实验项，不阻塞主流程
- 当前项目已升级为 2.5D 蝴蝶 rig：body、前翅、后翅分层驱动
- 当前真实素材先通过同源整图切片进入 rig，正式分层素材后续可直接替换
- 当前捕获循环已改为：
  - 搜索隐藏信号
  - 雷达锁定
  - 蝶影显形
  - 压圈追踪
  - 时机撒网
- 当前飞行拟态会表现出：
  - 悬停时的微漂浮和轻抖动
  - 转向时的 body 倾斜与 bank
  - 左右翅不同步的开合
  - 擦网或失手后的受惊逃逸
  - 捕捉成功前的短时失稳

## 正式素材替换说明

当前代码已经预留了“可替换素材”的结构，目前默认使用 **同源整图切片** 跑 2.5D rig。  
如果你后续继续补正式分层素材，推荐使用 **PNG 透明底**。

### 推荐规格

- 单只蝴蝶图
- 透明背景
- 建议尺寸：`512x512` 到 `1024x1024`
- 主体尽量居中
- 四周保留少量透明边距
- 2-3 种蝴蝶风格保持一致

### 正式分层素材替换方式

当前 rig 支持两种思路：

1. `sliced rig`
   - 用单张整蝶图切出 body / 左前翅 / 右前翅 / 左后翅 / 右后翅
   - 当前项目默认就是这种模式
2. `layered rig`
   - 使用正式分层素材直接替换
   - 推荐准备：
     - `body`
     - `left-wing-front`
     - `right-wing-front`
     - `left-wing-back`
     - `right-wing-back`

正式替换素材后，优先调整：

- `size`
- `captureRadius`
- `motion`
- rig 的切片与 pivot 参数

### 代码接入方式

`app.js` 中的 `butterflyCatalog` 是蝴蝶配置表。  
后续替换素材时，优先改这里：

- `assetSrc`
- `size`
- `captureRadius`
- `signalDifficulty`
- `trackHoldMs`
- `timingWindowSize`
- `timingCursorSpeed`
- `alertMax`
- `falseSignalChance`
- `palette`
- `motion`
- rig 素材的切片与 pivot 参数

## 人工验证建议

### 核心闭环

- 页面能正常打开
- 相机可请求并启动
- 至少有 2-3 种蝴蝶能正常出现
- 开始后不会直接可抓，必须先经过雷达搜索与锁定
- 不按住搜索或锁得不够准时，锁定条不会无条件上涨
- 蝴蝶飞行具有明显的扇翅、漂移、转向和悬停感
- 转向时能看出 body 倾斜和左右翅幅度差异
- Rare / Epic 的信号稳定度、压圈时长、出手窗口大小有明显差异
- 蝶影锁定后会先经历短暂显形，再进入压圈追踪
- 压圈完成后会打开撒网时机条
- 擦网未中与 Miss 都会提高警觉，并可能触发逃逸
- 测试网耗尽时会直接结束本轮
- 擦边未命中时能观察到短时受惊逃逸
- 成功捕捉前能看到短时失稳
- 成功后结果卡片正常展示
- 点击“继续探索”后可进入下一轮

### 实验模式

- 动作权限被拒绝时，页面仍能正常使用雷达主流程
- 动作权限允许后，切到甩动模式不影响搜索和压圈环节
- 将蝴蝶压入准星后甩动，可作为次级实验捕捉方式

### Safari 手测步骤

1. 用 iPhone Safari 打开 GitHub Pages 页面
2. 点击“开始体验”，允许相机权限
3. 按住屏幕拖动准星，确认需要先找到强信号并锁满后才会显形
4. 显形后观察悬停、普通巡航、转向时的开合、倾斜和 bank
5. 做一次擦网或 Miss，确认警觉值上涨且会触发受惊逃逸
6. 再做一次成功捕捉，确认压圈、撒网条和结果揭晓前的短时失稳都正常
7. 切到实验甩动模式，确认主流程和结果页没有被新捕获 loop 破坏

## 已知限制

- 不建议直接用 `file://` 打开；相机通常要求安全上下文
- iPhone Safari 的动作权限必须由用户手势触发
- 当前是 Safari-first 的 2.5D rig 拟态，不是真实纪录片级生物仿真
- 当前测试网仍是固定 5 次，分享 / 邀请 / 修复补网只留了后续扩展钩子
- 目前仍是单页静态 Demo，没有后台、账号、地图或活动系统
