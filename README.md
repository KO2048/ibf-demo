# ifafa Demo

这是一个面向 **iPhone Safari** 的轻量静态前端 Demo，用来验证单机版“发现 - 捕捉 - 揭晓”体验闭环。

当前版本只验证本地可运行的核心玩法，不涉及地理位置或 geofence：

- 相机背景
- 2-3 种蝴蝶变体
- 更自然的扇翅与拟态飞行
- 滑动捕捉主流程
- 甩动捕捉实验模式
- 捕捉结果反馈与继续探索

## 文件

- `index.html`
- `styles.css`
- `app.js`

## 使用方式

1. 把整个目录部署到 **HTTPS** 静态托管
   - GitHub Pages
   - Netlify
   - Vercel
   - Cloudflare Pages
2. 用 **iPhone Safari** 打开页面
3. 点击“开始体验”
4. 允许相机权限
5. 观察蝴蝶飞行轨迹，快速滑动穿过蝴蝶完成捕捉
6. 如需测试实验模式，再点击“实验模式：甩动”并允许动作权限

## 当前体验重点

- 默认演示路径是 **滑动捕捉**
- 甩动模式是次级实验项，不阻塞主流程
- 蝴蝶飞行不再是机械直线移动，而是带有：
  - 连续扇翅
  - 上下漂移
  - 随机转向
  - 短暂停顿 / 悬停
  - 边界回避

## 正式素材替换说明

当前代码已经预留了“可替换素材”的结构，默认使用内置占位蝴蝶造型。  
如果你后续要提供正式蝴蝶素材，推荐使用 **PNG 透明底**。

### 推荐规格

- 单只蝴蝶图
- 透明背景
- 建议尺寸：`512x512` 到 `1024x1024`
- 主体尽量居中
- 四周保留少量透明边距
- 2-3 种蝴蝶风格保持一致

### 更高优先级素材

如果你能提供下面这些，飞行动画会更自然：

1. 左翅 / 右翅 / 身体分层 PNG
2. 或至少提供张开 / 收拢两帧
3. 如果只有单张整蝶 PNG，也可以先替换为整图，再通过整体缩放和旋转做近似扇翅

### 代码接入方式

`app.js` 中的 `butterflyCatalog` 是蝴蝶配置表。  
后续替换素材时，优先改这里：

- `assetSrc`
- `size`
- `captureRadius`
- `palette`
- `motion`

如果 `assetSrc` 提供了 PNG 路径，代码会切换到图片渲染模式；如果为空，则继续使用内置占位造型。

## 人工验证建议

### 核心闭环

- 页面能正常打开
- 相机可请求并启动
- 至少有 2-3 种蝴蝶能正常出现
- 蝴蝶飞行具有明显的扇翅、漂移、转向和悬停感
- 滑动快速掠过蝴蝶时可完成捕捉
- 成功后结果卡片正常展示
- 点击“继续探索”后可进入下一轮

### 实验模式

- 动作权限被拒绝时，页面仍能正常使用滑动主流程
- 动作权限允许后，切到甩动模式可看到中央锁定圈
- 将蝴蝶引入中央区域后甩动，能触发实验捕捉

## 已知限制

- 不建议直接用 `file://` 打开；相机通常要求安全上下文
- iPhone Safari 的动作权限必须由用户手势触发
- 当前是“轻写实 2D”飞行表现，不是真实生物级仿真
- 目前仍是单页静态 Demo，没有后台、账号、地图或活动系统
