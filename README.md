# ifafa Demo

这是一个面向 **iPhone Safari** 的轻量静态前端 Demo，用来验证单机版“发现 - 捕捉 - 揭晓”体验闭环。

当前版本只验证本地可运行的核心玩法，不涉及地理位置或 geofence：

- 相机背景
- 3 种真实蝴蝶素材变体
- 5 层 2.5D 蝴蝶 rig
- 常驻画框 + timing 条
- 甩动捕捉主流程
- 滑动备用兜底
- 捕捉结果反馈与继续探索

## 使用方式

1. 把整个目录部署到 **HTTPS** 静态托管
   - GitHub Pages
   - Netlify
   - Vercel
   - Cloudflare Pages
2. 用 **iPhone Safari** 打开页面
3. 点击“开始体验”
4. 允许相机权限
5. 点击“主玩法：甩动”并允许动作权限
6. 将蝴蝶引入中央画框，等待 X 落入高亮窗口 MN
7. 在窗口内快速甩动完成命中
8. 如需兜底测试，再切回“备用：滑动”

## 当前体验重点

- 默认演示路径是 **入框 + timing + 甩动**
- 滑动保留为备用兜底，不再是主玩法
- 当前项目默认使用 `sliced rig`：把单张真实 PNG 切成 body / 前翅 / 后翅 5 层来驱动动作
- 当前 timing 规则是：
  - `AB` 为整条 timing 条
  - `MN` 为有效命中窗口
  - `X` 在 `AB` 上以固定速度左右往返
  - 只有蝴蝶在中央画框内，且甩动时 `X ∈ MN`，才命中
- 当前稀有度只影响 `MN` 长度：
  - Common 最宽
  - Rare 次之
  - Epic 最窄
- 当前 `q` 统一，未来再扩展更好的网让 `q` 更慢
- 当前飞行拟态重点是：
  - 悬停时的微漂浮和低速拍翅
  - 巡航时可见的前后翅开合
  - 转向时的 body 倾斜和左右翅不对称
  - 擦边未命中后的受惊逃逸
  - 命中成功后的短时失稳

## 正式素材替换说明

当前代码已经预留了“可替换素材”的结构，目前项目中已经接入 3 张真实蝴蝶 PNG。  
如果你后续继续补新蝴蝶素材，推荐使用 **PNG 透明底**。

### 推荐规格

- 单只蝴蝶图
- 透明背景
- 建议尺寸：`512x512` 到 `1024x1024`
- 主体尽量居中
- 四周保留少量透明边距
- 2-3 种蝴蝶风格保持一致

### 素材模式

当前代码支持 3 种素材模式：

1. `sliced rig`
   - 用单张整蝶 PNG 切出 body / 左前翅 / 右前翅 / 左后翅 / 右后翅
   - 当前项目默认就是这种模式
2. `placeholder rig`
   - 当正式 PNG 不可用时，回退到内置占位造型
3. `layered rig`
   - 使用正式分层素材直接替换切片来源

### 更高优先级素材

如果你能提供下面这些，飞行动画会更自然：

1. `body`
2. `left-wing-front`
3. `right-wing-front`
4. `left-wing-back`
5. `right-wing-back`

### 代码接入方式

`app.js` 中的 `butterflyCatalog` 是蝴蝶配置表。  
后续替换素材时，优先改这里：

- `rigMode`
- `assetSrc`
- `size`
- `captureRadius`
- `sliceBoxes`
- `rig.frontPivot`
- `rig.backPivot`
- `palette`
- `motion`

如果你继续升级成正式分层素材，优先保留当前 pivot 与 motion 参数，再按单只蝴蝶微调。

## 人工验证建议

### 核心闭环

- 页面能正常打开
- 相机可请求并启动
- 至少有 2-3 种蝴蝶能正常出现
- 蝴蝶飞行能看出明确的开合、转身、受惊和捕获前失稳
- 中央画框常驻显示
- timing 条全程持续往返
- 蝴蝶入框且 `X ∈ MN` 时甩动能命中
- 蝴蝶入框但 `X ∉ MN` 时甩动不能命中
- 蝴蝶不在框内时甩动也不能命中
- 成功后结果卡片正常展示
- 点击“继续探索”后可进入下一轮

### 备用路径与容错

- 动作权限被拒绝时，页面仍能正常使用滑动主流程
- 切到“备用：滑动”后，仍可用原有滑动方式完成捕捉
- 甩动 miss 后蝴蝶只会受惊偏移，不直接结束本轮

## 已知限制

- 不建议直接用 `file://` 打开；相机通常要求安全上下文
- iPhone Safari 的动作权限必须由用户手势触发
- 当前是“轻写实 2D”飞行表现，不是真实生物级仿真
- 目前仍是单页静态 Demo，没有后台、账号、地图或活动系统
